## Important Note About CMDB IDs

**STI codes are typically the same as CMDB IDs.** For example:
- STI: `WIND-002` = CMDB ID: `WIND-002`
- STI: `OSL-001` = CMDB ID: `OSL-001`

However, to get the definitive CMDB ID mapping, you would need to:
1. Query **ServiceNow CMDB** directly (authoritative source)
2. Access Jira custom fields (may require special permissions)
3. Check individual issue descriptions

   - Export CMDB ID field
   - Match with SOAR issues

2. **Use Jira API with Custom Fields:**
   - Query with `fields=*all` to get custom fields
   - Look for CMDB-related custom fields
   - Extract CMDB IDs from those fields

3. **Export from Jira with All Fields:**
   - Export all SOAR issues (not just open)
   - Include all custom fields
   - Parse CMDB IDs from the export
